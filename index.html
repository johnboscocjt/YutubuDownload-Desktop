Downloader Power in Layman Language

Think of this like everyday stuff combined together:

  • Bash orchestrator engine: acts like the trip manager, combining all helper tools so one download flow starts and finishes reliably.
  • Cookie service with shared cookie file and lock: prepares trusted access and coordinates sessions so the tool can pass platform checks automatically.
  • Adaptive progress controller: detects weak network conditions and changes display mode while keeping the download running instead of freezing.
  • Quality resolver on top of yt-dlp formats: picks the requested quality first, then safely falls back to the nearest working quality if needed.
  • Session isolation plus retry/resume strategy: prevents collisions and recovers interrupted transfers, giving consistent results with fewer failed attempts.

Technical Explanation

  • Bash orchestration over yt-dlp + ffmpeg.
  • Deno-first JS challenge runtime, Node fallback.
  • Shared `cookies.txt` with lock-based refresh for concurrency safety.
  • Session-aware temp/log paths to avoid cross-instance collisions.
  • Adaptive UI: single-line detailed bar by default, low-network mode only when truly degraded.

How To Use

  1. Run ytd
  2. Paste URL
  3. Choose single or playlist
  4. Choose video or MP3
  5. Select quality and confirm

Platform Notes

  • Linux: fully supported.
  • Windows: use WSL or a Linux VM.
  • macOS: supported because it is Unix based.
  • Distro-agnostic design across Unix-like systems.

Network Behavior

  • Good network: full ETA + speed progress.
  • Very weak network: low-network adaptive mode.
  • Low-network mode means still downloading, not failing.
